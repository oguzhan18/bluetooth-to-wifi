# Security policy

## Supported versions

Security fixes are applied to the default branch (`main` / `master`) of this repository. Tag releases when publishing packages.

## Reporting a vulnerability

Please **do not** open a public GitHub issue for undisclosed security problems.

- Open a **private** security advisory on GitHub (repository → Security → Advisories), or  
- Contact the maintainers with a clear subject line and reproduction steps.

Include: affected component (Python / Node / ESP32), version or commit, impact, and suggested fix if any.

## Scope

- Out of scope: physical BLE radio jamming, compromised host OS, or misconfigured credentials on your devices.

## Disclosure

We aim to acknowledge reports within a few business days and coordinate a fix and release timeline before public disclosure.
